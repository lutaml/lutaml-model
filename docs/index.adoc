== Learning Paths

[[beginner-path]]
=== Beginner Path (Getting Started)

Follow this path to learn Lutaml::Model fundamentals in order:

1. **Installation & Setup**
   * link:../README.adoc#installation[Installation] - Get Lutaml::Model running
   * link:serialization_adapters.adoc[Serialization Adapters] - Configure adapters for your formats

2. **Core Concepts**
   * link:models.adoc[Models] - Define your first model class
   * link:attributes.adoc#attribute-definition[Basic Attributes] - Add attributes to models
   * link:value_types.adoc#value-definition[Value Types] - Understand built-in types

3. **First Serialization**
   * link:serialization_keyvalue.adoc#mapping[JSON/YAML Mapping] - Serialize to key-value formats
   * link:serialization_xml_mapping.adoc#mapping-xml[XML Mapping Basics] - Serialize to XML

4. **Essential Features**
   * link:collections.adoc#collection-definition[Collections] - Work with arrays of models
   * link:validation.adoc[Validation] - Validate your data

**Estimated Time**: 2-3 hours

[[intermediate-path]]
=== Intermediate Path (Building Applications)

Master these topics after completing the beginner path:

1. **Advanced Attributes**
   * link:attributes.adoc#polymorphic-attributes[Polymorphic Attributes] - Handle multiple types
   * link:attributes.adoc#collection-attributes[Collection Attributes] - Arrays and ranges
   * link:attributes.adoc#derived-attributes[Derived Attributes] - Computed values
   * link:attributes.adoc#choice-attributes[Choice Attributes] - Min/max constraints

2. **Advanced Serialization**
   * link:serialization_format_independent.adoc#mapping-value-transform[Value Transformations] - Transform data during serialization
   * link:serialization_advanced_mapping.adoc[Advanced Mapping] - Custom methods and delegation
   * link:serialization_missing_values.adoc[Missing Values] - Handle nil/empty/omitted properly

3. **XML Mastery**
   * link:serialization_xml_namespaces.adoc[XML Namespaces] - Multi-namespace documents
   * link:serialization_xml_mapping.adoc#mixed-content[Mixed Content] - Rich text support
   * link:serialization_xml_mapping.adoc#ordered-content[Ordered Content] - Element sequencing

4. **Collections & Data Structures**
   * link:collections.adoc#keyed-collections[Keyed Collections] - Hash-like collections
   * link:serialization_collections.adoc[JSONL & YAML Stream] - Multi-document formats
   * link:serialization_keyvalue.adoc#collection-keyed-by-value[Root Mappings] - Complex keyed structures

**Estimated Time**: 6-8 hours

== Quick Reference Tables

[[attr-options]]
=== Attribute Options Reference

[cols="1,2,3",options="header"]
|===
| Option | Values | Description

| `collection:`
| `true`, `Range` (e.g., `1..`, `0..1`, `5..10`)
| Define attribute as array with optional cardinality

| `default:`
| `-> { value }` (lambda)
| Default value for attribute

| `values:`
| `[value1, value2, ...]` (array)
| Restrict to specific enumeration values

| `pattern:`
| `/regex/` (regexp)
| Validate string values against pattern

| `required:`
| `true`, `false`
| Mark attribute as required (validates presence)

| `polymorphic:`
| `true`, `[Class1, Class2]` (array)
| Accept subclass instances

| `transform:`
| `{ export:, import: }` (hash of procs)
| Transform values during serialization

| `method:`
| `:method_name` (symbol)
| Derive value from instance method

| `render_default:`
| `true`, `false`
| Force rendering of default values

| `render_nil:`
| `:as_empty`, `:as_nil`, `:omit`
| Control how nil values are rendered

| `render_empty:`
| `:as_empty`, `:as_nil`, `:omit`
| Control how empty values are rendered

| `raw:`
| `true`, `false`
| Read XML content as raw string (XML only)

| `initialize_empty:`
| `true`, `false`
| Initialize collections as `[]` instead of `nil`

|===

[[mapping-methods]]
=== Mapping Methods Reference

==== XML Mapping Methods

[cols="1,3",options="header"]
|===
| Method | Description

| `root "name"`
| **DEPRECATED** Use `element` instead. Set root element name

| `element "name"`
| Set element name for this model/type

| `no_root`
| **DEPRECATED** Omit for type-only models. Mark model as type-only (no root element)

| `namespace`
| Declare XML namespace for element/type

| `namespace_scope`
| Consolidate namespace declarations at root

| `mixed_content`
| Enable mixed content mode (text + elements)

| `sequence do ... end`
| Enforce strict element ordering

| `map_element "name", to: :attr`
| Map XML element to attribute

| `map_attribute "name", to: :attr`
| Map XML attribute to attribute

| `map_content to: :attr`
| Map element text content to attribute

| `map_all to: :attr`
| Capture entire XML as raw string

|===

==== Key-Value Mapping Methods

[cols="1,3",options="header"]
|===
| Method | Description

| `map "key", to: :attr`
| Map key to attribute (JSON/YAML/TOML/Hash)

| `map_all to: :attr`
| Capture entire content as raw string

| `map to: :attr, root_mappings: {...}`
| Map keyed collections with key extraction

| `map "key", to: :attr, child_mappings: {...}`
| Extract nested values from key-value data

| `map "key", to: :attr, delegate: :model`
| Delegate mapping to nested object

|===

==== Collection Mapping Methods

[cols="1,3",options="header"]
|===
| Method | Description

| `instances :name, Type`
| Define collection attribute and type

| `map_instances to: :attr`
| Map collection instances (key-value/JSONL/YAMLS)

| `ordered by: :attr, order: :asc`
| Define collection ordering

| `root "name"`
| Set collection container name

| `no_root`
| Omit container (root-level collection)

|===

| `""` or `[]`

| `nil`
